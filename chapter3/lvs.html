LVS的讲解和实践

LVS

LVS是Linux Virtual Server的简称。作为Linux内核的一员, 主要是基于netfilter实现的四层负载均衡集群系统，可在传输层将一组服务器构成一个实现可伸缩的、高可用网络服务的虚拟服务群集。

在本章节将从Linux netfilter的流程讲起，围绕LVS的构成、工作模式以及操作实践等相关内容的撰写。

需要指出的是：在上产环境中仅单纯利用LVS解决高并发场景的负载均衡已经很少见，比较流行的是基于DPDK+LVS的改进方案，在后续篇幅中也将继续讲解这部分的内容。

LVS的组成

LVS由两部分组成：ipvs和ipvsadm

  • ivps(ip virtual server)：Linux内核的一部分，实现传输层调度、转发等负载均衡的核心功能。
  • ipvsadm：ivps在应用层的命令接口，负责为ivps定义规则、集群服务等 （需单独安装）

LVS的一些概念性名词
CIP Client IP 客户端的IP
VIP Virtual IP LVS实例IP，一般是暴露在公网中的地址；向外部直接面向用户请求，作为用户请求的目标的IP地址
DIP Director IP 主要用于和内部主机通讯的IP地址
RIP Real IP 后端服务器的真实IP
DS Director Server 指的是前端负载均衡器节点
RS Real Server 后端真实的工作服务器
