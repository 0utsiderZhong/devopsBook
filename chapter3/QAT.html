使用Intel® QAT优化

使用Intel® QAT优化请求：SSL及zlib压缩优化

QAT(Quick Assist Technology )是Intel公司推出的一种专用硬件加速卡。

QAT不仅能对SSL非对称加解密算法（RSA、ECDH、ECDSA、DH、DSA等）具有加速，而且提供了对其原生zlib与QAT适配的Zlib-SHIM接口，对数据的压缩与解压也具有很好的加速效果。QAT驱动采用UIO（Userspace I/O）技术，其大部分处于用户态、只有少部分处理硬件中断应答等逻辑处于内核态，这样不仅方便用户调试，而且还解决了内核不支持浮点数运算的问题。

Intel® QAT的硬件设备性能

以Intel® QuickAssist Adapter 8970为例，能看出针对加解密、压缩以及解压缩等运算都有非常高的性能指标。

intel® QuickAssist Adapter 8970
批量加密性能 103Gbps@4KB Packet
RSA2K Key 解密性能 100k ops/s
动态压缩 66Gbps@64KB
动态解压缩 160Gbps@64KB

在Intel的官网，对于如何使用QAT以及设置HTTPS进行完整的描述和安装指导，我结合ZLIB的设置，整理出以下内容，以供读者参考

文章内使用的 synch_mode_nginx 是 Nginx 经过优化的版本，英特尔将其用于支持 Nginx 硬件和软件加速。下载之后可直接编译。相关说明请参阅以下链接：https://github.com/intel/asynch_mode_nginx

https://www.intel.cn/content/www/cn/zh/developer/articles/guide/nginx-https-with-qat-tuning-guide.html

适用于 OpenSSL 的英特尔 QAT 引擎链接：https://github.com/intel/QAT_Engine

Intel® QAT安装和设置

Intel® QuickAssist Accelerator 是一块PCIe卡，首先需要将其安装在服务器的PCIe插槽中。以下为QAT加速卡的图片

卡安装完毕后，需要确认BIOS设置，将PICe link speed设置到 Gen4,X16 以获得最好的性能。

驱动安装

用户可以从以下网址下载加速卡驱动，解压缩并进行安装操作。本文以/QAT为驱动安装目录

https://www.intel.com/content/www/us/en/download/19734/intel-quickassist-technology-driver-for-linux-hw-version-1-7.html

cd /
mkdir QAT

将QAT.L.4.20.0-00001.tar.gz 拷贝至 /QAT 目录

cd /QAT
tar xvzf ./ QAT.L.4.20.0-00001.tar.gz
./configure --enable-qat-lkcf --enable-icp-dc-sym-only --enable-kapi
make
make install
make samples-install

在build下， 会出 cpa_sample_code.ko 模块，可以用 insmod ./build/cpa_sample_code.ko 加载cpa_sample_code执行测试

驱动安装完毕后，命令输出中可以看到有3个QAT加速设备(qat_dev)处于启动状态, 然后用如下命令启动QAT服务

service qat_service start

OpenSSL 和 QAT_Engine安装

QAT用到的 openssl 的版本需要 1.1.0 之上的, 进行OpenSSL升级

OpenSSL源码下载

cd ~
wget https://www.openssl.org/source/openssl-1.1.1s.tar.gz

OpenSSL编译安装 (本文安装在/usr/local/ssl中)

tar xvzf openssl-1.1.1s.tar.gz
cd ./openssl-1.1.1s
./config --prefix=/usr/local/ssl -W1,-rpath,/usr/local/ssl/lib
make
make install

// 添加动态库
echo /usr/local/ssl/lib/ > /etc/ld.so.conf.d/qat.conf
ldconfig

/usr/local/ssl/bin/openssl version //进行验证

QAT Engine安装

QAT Engine源码下载

export OPENSSL_ENGINES=/usr/local/ssl/lib/engines-1.1
cd /usr/local/

git clone https://github.com/intel/QAT_Engine.git

QAT Engine编译安装

cd ./QAT_Engine
./autogen.sh
./configure --with-qat_dir=/QAT --with-openssl_install_dir=/usr/local/ssl
make && make install

QAT服务配置文件替换

为了使得QAT Engine能和 QAT Service联合工作，还需要把QAT Engine目录下的实例配置拷贝至/etc目录下，替换掉QAT原来的配置文件

cd /usr/local/QAT_Engine/qat/config/c6xx/multi_process_event-driven_optimized
cp * /etc/

替换文件后，重启qat_service以加载新的配置文件

service qat_service restart

Nginx 安装及优化设置

Intel QAT 加速卡可以对HTTPS的请求进行异步请求， 加快证书处理， 降低系统性能消耗。Nginx作为代理，可以代理HTTPS请求，需要重新编译，支持QAT加速卡，这样才能将请求给QAT加速卡。

Nginx以及相关的Path下载

wget http://nginx.org/download/nginx-1.18.0.tar.gz
git clone https://github.com/intel/asynch_mode_nginx.git
git clone https://github.com/intel/QATzip.git

Nginx Path

tar -zxf nginx-1.18.0.tar.gz
diff -Naru -x .git nginx-1.18.0 asynch_mode_nginx > async_mode_nginx_1.18.0.patch
cd nginx-1.18.0 
patch -p1 < ../async_mode_nginx_1.18.0.patch

编译 QATzip

export QZ_ROOT=/home/QATzip
echo 1024 > /sys/kernel/mm/hugepages/hugepages-2048kB/nr_hugepages
rmmod usdm_drv
insmod $ICP_ROOT/build/usdm_drv.ko max_huge_pages=1024 max_huge_pages_per_process=16
cd $QZ_ROOT
./configure --with-ICP_ROOT=$ICP_ROOT
make clean
make all install
./setenv.sh

/etc/init.d/qat_service restart
systemctl restart qat_service

QATzip测试验证

cd $QZ_ROOT/test/performance_tests
./run_perf_test.sh

编译Nginx

cd /home/nginx-1.18.0
 export NGINX_INSTALL_DIR=/usr/local/nginx
./configure \
    --prefix=$NGINX_INSTALL_DIR \
    --with-http_ssl_module \
    --add-dynamic-module=modules/nginx_qatzip_module \
    --add-dynamic-module=modules/nginx_qat_module/ \
    --with-cc-opt="-DNGX_SECURE_MEM -I$OPENSSL_LIB/include -I$ICP_ROOT/quickassist/include -I$ICP_ROOT/quickassist/include/dc -I$QZ_ROOT/include -Wno-error=deprecated-declarations" \
    --with-ld-opt="-Wl,-rpath=$OPENSSL_LIB/lib -L$OPENSSL_LIB/lib -L$QZ_ROOT/src -lqatzip -lz" 
make 
make install

Nginx 优化设置

load_module modules/ngx_http_qatzip_filter_module.so;
load_module modules/ngx_ssl_engine_qat_module.so;

ssl_engine {
    use_engine qatengine;
    default_algorithms RSA,EC,DH,DSA;
    qat_engine {
        qat_offload_mode async;
        qat_notify_mode poll;
        qat_poll_mode heuristic;
        qat_sw_fallback on;
    }
}

评测结果

由于作者没有QAT卡，测试以及安装等实践很难进行，我在一个阿里社区看到Tengine（阿里7层流量入口）一个QAT实践文章，在这里引用该文章的测试数据，以便读者了解性能提升情况

数据来源 https://developer.aliyun.com/article/597750

QAT 对 SSL 加速前后系统CPU对比

  • RSA套件提升3.8倍(8核时)
  • ECDHE-RSA提升2.65倍(8核时)
  • ECDHE-ECDSA(P-384) 提升2倍(16核时)
  • ECDHE-ECDSA(P-256) 8核达到QAT硬件处理峰值16k左右， 只有23%的性能提升。

QAT 对 Gzip 加速前后系统CPU对比

当qps为10k左右时Tengine Gzip使用QAT加速后CPU节省15%左右，且Gzip基本上完全卸载、随着其占比变高，优化效果将越好。（在2017年双11零点流量峰值，Tengine加 速机器相比普通机器性能提升 21%。 ）
