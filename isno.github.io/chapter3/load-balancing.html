负载均衡概述

负载均衡概述

负载均衡是基于横向扩展保证高并发系统架构高可用的优化方法。

常用的软件负载均衡有基于TCP/IP四层、和七层方案。在域名解析上，也可以用DNS做负载均衡，不过由于比较明显的缺点，实际上已经很少使用。

四层负载均衡是IP数据包在kernel中进行路由分发的软件方案，比如LVS，在Linux内核中利用netfilter对IP包就行修改转发，以实现流量分发的负载均衡目标， 这类方案无法识别上层协议，灵活性不高，但性能指标较好，一般用pps(包转发/秒)衡量性能。

在七层的应用层，则使用Nginx此类的方向代理方案，在Nginx中可以识别流量的协议，配合Lua，如OpenResty，可以实现功能丰富且性能较高的网关方案。 Nginx使用epoll相关的技术实现高并发的支撑，一般用qps(请求/秒)衡量性能，可以实现c10k的并发处理。

扩展

常用的网络应用架构设计，一般采用上游DNS指派分区服务器集群，然后服务器集群采用LVS + Nginx 的负载均衡方案。

实际基于普通的LVS的方案有一些缺点，随着互联网规模的发展，也要开始考虑c10m的问题，

在目前的高并发需求下，服务器启用linux 的NAPI驱动的polling来缓解大量的网卡中断基本是必须的，像LVS基于内核netfilter这种方式，在用户态和内核态的交互中，会有大量像Context Switch、Page switch、 Cache Miss，流水线冲刷的消耗，这些都会制约IP包转发的性能。

改进及解决

目前，很多企业如Facebook的Katran、美团的MGW、爱奇艺的DPVS等都使用Intel 提供的DPDK套件进行kernel baypass，直接全部在用户态进行流量的处理，基于爱奇艺的DPVS进行测试，相较普通的LVS转发方案，pps转发能力可以提升六倍有余，这种方案也使得C10M的解决成为现实。
